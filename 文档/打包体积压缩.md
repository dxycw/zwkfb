# 打包体积压缩

**本项目所有使用的打包体积压缩：**

## 一、打包分包

### Multiplatform平台：

* 暂无

### Android平台：

在项目的 `androidApp`的`build.gradle.kts` 文件中使用“splits”配置分包，如下配置：

```kotlin

android{
    //...
    // ABI分割
    splits {
        abi {
            isEnable = true // 是否开启ABI分割
            reset()  // 清空默认的abi配置
            include("armeabi-v7a", "arm64-v8a", "x86", "x86_64") // 指定要支持的ABI
            isUniversalApk = false     // 是否生成通用apk
        }
    }
    //...
}

// 修改输出文件名
androidComponents.onVariants { variant ->
    var versionName = "0.0.1"
    // 获取当前 APK 对应的 ABI（架构）
    var abi = "universal"
    // 构建时间
    val timeStamp = SimpleDateFormat("yyyyMMdd_HHmm", Locale.getDefault()).format(Date())
    variant.outputs.forEach { output ->
        if (output is VariantOutputImpl) {
            versionName = output.versionName.get()
            // 获取当前 APK 对应的 ABI（架构）
            output.filters.forEach { abi = it.identifier }
            // 修改输出文件名
            output.outputFileName = "应用名-${versionName}-${abi}-${timeStamp}.apk"
        }
    }
}

```

## 三、去掉未使用的依赖库


### Multiplatform平台：

在项目的 `shared`或`core`的`build.gradle.kts` 文件中添加用不到的依赖项：

```kotlin
dependencies {
    implementation("com.github.dxycw.zwkfb:zwkfb:$最新版本"){

        // Multiplatform平台：
        // 如果使用图标，请把material-icons-extended依赖库注释掉
        exclude("org.jetbrains.compose.material", "material-icons-extended")

        // Android平台：
        exclude("org.jetbrains.kotlinx", "kotlinx-coroutines-android")

        // Desktop (Windows, MacOS, 和 Linux)平台： 
        // JNA 核心库
        exclude("net.java.dev.jna", "jna")
        // JNA Platform（包含 Windows API、POSIX 等封装）
        exclude("net.java.dev.jna", "jna-platform")
        
    }
}
```

### Android平台：
